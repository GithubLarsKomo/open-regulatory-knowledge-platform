# SPEC-WritingRules.md

## Normative Language

Use:

- shall = mandatory
- should = recommended
- may = optional
- must not = prohibited

## Required Sections for Detail SPECs

Each detail SPEC shall contain:

1. Purpose
2. Scope
3. Stakeholders
4. Domain Model
5. Requirements
6. Interfaces
7. Data Model
8. Workflow
9. Security
10. AI Support
11. Acceptance Criteria
12. Open Questions

## Style Rules

- Every requirement shall have an ID.
- Requirements shall be testable.
- Avoid implementation details unless the file is explicitly implementation-specific.
- Separate business logic from storage technology.
